## 所有电脑的交互方式：
* 图形界面（GUI）：用鼠标点击、拖拽文件夹、双击程序。这是给普通用户用的，直观但慢。也就是我此前一直用的
* 命令行界面（CLI）：用键盘打字，电脑执行命令。这是给开发者用的，快但需要学语法。很多操作图形界面根本做不到，或者做起来极慢。比如"把这个文件夹里所有 .py 文件都找出来，统计行数"，鼠标一个一个点要半小时，命令行一行代码1秒完成。另外服务器根本没有图形界面，只有通过SSH远程输入命令行。而系统命令语言就是CLI的语言。

## 关键概念
* Terminal / Shell：黑框框（可以叫terminal或shell），包在操作系统外面，是你和操作系统内核之间的翻译层。你（打字）→ Shell / terminal → 操作系统内核 → 执行 → 结果返回给你。
	* Windows上的shell有三种：
		* cmd：最老，功能弱，语法是windows自己发明的
		* powershell：2006年推出，功能强，语法还是windows风格
		* git bash：第三方shell，非windows自带，需要安装，说的是linux语言
* Shell脚本语言泛指所有用来操作命令行的语言，具体包括：
	* **Bash**（Bourne Again Shell）：这是具体的一种 Shell，是Linux语言的正式称呼。**日常对话中Shell脚本基本指的就是Bash，因为全世界的开发者教程、GitHub 项目文档、AI 开发框架的安装说明，99% 写的都是 Linux 语法。**
	- **Zsh**（Z Shell）：另一种 Shell，Bash 是1989年出现的，是 Linux 世界的默认标准，全世界服务器几乎都是 Bash。Zsh 是 Bash 的升级版，2019年开始成为 Mac 的默认 Shell，99%和bash语法相同。
	- **PowerShell**，微软自己的 Shell
	- **CMD** ，Windows 最老的 Shell（严格说它的语言叫 batch 脚本）

## bash语言
* `LPR18@LILLARDLIN-NB1 MINGW64 /d/Projects` ，这是提示符Prompt，是当前定位信息
	* `LPR18` 当前登录 Windows 系统的用户名
	* `LILLARDLIN-NB1` 我的电脑名称，若SSH远程连接到服务器时，这里会变成服务器的名字
	* `MINGW64` shell的名称，Min代表 Minimalist，GW代表 GNU（GNU 是 Linux 系统核心工具集的名称） for Windows，64代表支持 64位 操作系统，即现在这个shell是gitbash
	* `/d/Projects` 现在的位置
* 文件相关
	* `cd` change directory，进入文件夹
	* `mkdir` make directory，创建文件夹，之后空格一次性可以写多个文件夹名字以创建，文件夹名字后以/结尾代表是文件夹
	* `touch` 创建一个空文件，之后空格一次性可以一次性创多个空文件，每个空文件以空格隔开，需要带文件后缀
	* `ls -la` 列出该文件夹中所有文件和文件夹

## Git命令
Git是一个开源**版本管理软件**，会在你文件夹里偷偷建一个小型数据库（就是 .git 隐藏文件夹），里面有 Git 自己的一套文件格式、索引、日志，用来Git 软件自己读、自己对比、自己记版本。我们只需要会Git命令，即可查看并管理历史上各个代码版本。
- `git init` 把当前文件夹初始化成一个 Git 仓库。即Git可以开始追踪这个文件夹里的文件变化
- `git add .` 把当前文件夹里所有文件加入**待提交区**。 `.` 代表当前文件夹里的全部内容，`.` 也可以改成单个文件名
- `git commit -m ""` 正式保存一次版本记录到**本地git仓库**。`-m` 后是这次提交版本的说明。
- `git push` 把这个版本push到对应自己的github上，第一次push是`git push -u origin main`
	- 之后每次更新只需要`git status` (查看现在哪些本地文件没有对准git仓库)+ `git add .` + `git commit` + `git push` 
- `git branch` 查看当前分支名字，分支的意思是为了增新功能，可以在主分支基础上增加分支来测试
	- `git branch -M main` 把当前分支名字改成 `main`（push到github上的必要代码步骤），main是当前项目的正式版本线，GitHub 页面默认展示的通常就是 `main` 分支上的代码
	- `git branch 分支名字` 新建新的分支，命名为“分支名字”
	- `git branch switch -c 分支名字` 新建并切换到“分支名字”
	- `git switch main` + `git merge 分支名字` 把分支合并回 main
- `git log --oneline` 查看过去所有版本，并带有每次的commit id
	- `git checkout commit_id` 查看但不修改过去某一版
	- `git revert commit_id` 查看并修改过去某一版，但保留历史
